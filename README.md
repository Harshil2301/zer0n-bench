## Overview

Zer0n-Bench is a **provenance-aware vulnerability dataset** comprising 10,050 synthetically generated vulnerability scenarios across Web Applications, Smart Contracts, and APIs. Each label is cryptographically anchored to the Avalanche blockchain, enabling tamper detection and full provenance auditing.

**Key Features:**
- **Multi-Agent Consensus Labels** — Generated by adversarial Red/Blue agent team (Gemini 2.0 Pro / 1.5 Pro)
- **Immutable Blockchain Provenance** — 10,050 SHA-256 hashes committed to Avalanche Fuji C-Chain
- **Human Validation** — n=1,000 stratified sample reviewed by 4 security experts (Fleiss' κ=0.76)
- **Multi-Domain Coverage** — Web2 (PHP, JS, Python, Java) and Web3 (Solidity) in unified schema

## Quick Start

```bash
# Clone the repository
git clone https://github.com/Harshil2301/zer0n-bench.git
cd zer0n-bench

# Install dependencies
pip install -r requirements.txt

# Quick validation (sample 10 entries)
python validation/validate_schema.py --manifest splits.json --sample 10
# Expected output: "SUCCESS: Sample 10 hashes validated successfully against Fuji testnet"

# Full validation (all 10,050 entries)
python validation/validate_schema.py dataset/zeron_bench_v1.0_full.json
# Expected output: "SUCCESS: All 10050 entries passed validation."
```

## Verifying Blockchain Proofs

Each entry's SHA-256 hash is permanently recorded on Avalanche:

```bash
# Automated verification
python validation/validate_schema.py dataset/zeron_bench_v1.0_full.json

# Manual verification
# 1. Open blockchain/blockchain_proofs.csv
# 2. Copy any avalanche_tx_id
# 3. Visit: https://testnet.snowtrace.io/tx/<TX_ID>
# 4. Confirm the data field matches the sha256_hash
```
